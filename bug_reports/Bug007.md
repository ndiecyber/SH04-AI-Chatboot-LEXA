## Title
Tidak Ada Validasi Ukuran File — Upload File Sangat Besar Diterima Tanpa Batas

## Severity
🟠 **High**

## Priority
🟠 **P2 — Fix in Next Sprint**

## Description

Sidebar Streamlit menampilkan `help="Maksimal ukuran 10MB"` sebagai teks informasi, namun tidak ada validasi programatik di sisi Streamlit maupun backend FastAPI yang menolak file melebihi batas ini. File PDF berukuran besar (>10MB, >100 halaman) diterima, diproses dengan `pypdf`, di-embed dengan SentenceTransformers, yang menyebabkan:
- Response time sangat lambat (>60 detik untuk PDF 50MB)
- Potensi OOM (Out of Memory) untuk file sangat besar
- Blocking server karena embedding bersifat sinkron

## Steps to Reproduce
1. Upload file PDF berukuran 25MB via sidebar.
2. Observasi: spinner berputar lebih dari 60 detik.
3. Server mungkin timeout atau OOM.

## Expected Result
HTTP 413 atau 400: "File terlalu besar. Maksimum 10MB."

## Actual Result
File diterima dan diproses tanpa batas ukuran.

## Recommendation

**Backend (documents.py):**
```python
MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB

@router.post("/upload-temp/{session_id}")
async def upload_temp_document(session_id: str, file: UploadFile = File(...)):
    contents = await file.read()
    if len(contents) > MAX_FILE_SIZE:
        raise HTTPException(
            status_code=413,
            detail=f"File terlalu besar ({len(contents)//1024//1024}MB). Maksimum 10MB."
        )
```

**Streamlit (app.py):**
```python
uploaded_file = st.file_uploader("Pilih file", type=["pdf","txt"], max_bytes=10*1024*1024)
```
