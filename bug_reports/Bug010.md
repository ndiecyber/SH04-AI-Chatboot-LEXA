## Title
Mengosongkan File Uploader Memicu Full Session Reset — Riwayat Chat Hilang

## Severity
🟡 **Medium**

## Priority
🟡 **P3 — Fix Before Production**

## Description

Ketika pengguna menghapus file dari file uploader (klik X pada file yang sudah dipilih), aplikasi mendeteksi `uploaded_file is None` dengan `indexed_files` yang masih berisi nama file. Kondisi ini memicu **full session reset**: sesi dihapus dan dibuat ulang di backend.

Akibatnya: **seluruh riwayat chat yang tidak berhubungan dengan dokumen juga hilang**. Pengguna yang secara tidak sengaja mengklik X pada file uploader akan kehilangan percakapan mereka.

## Steps to Reproduce
1. Mulai percakapan 10 turn.
2. Upload file PDF.
3. Klik X pada file uploader untuk membatalkan upload.
4. Observasi: seluruh riwayat chat hilang, halaman dimuat ulang.

## Expected Result
Hanya memori dokumen sementara yang dibersihkan. Riwayat chat tetap ada.

## Actual Result
Full session reset — riwayat chat hilang.

## Recommendation

Pisahkan logika pembersihan dokumen temp dari reset sesi:

```python
# JANGAN hapus seluruh sesi — hanya hapus dokumen dari pipeline
if "indexed_files" in st.session_state and st.session_state.indexed_files:
    try:
        # Endpoint baru: hanya bersihkan pipeline sesi tanpa hapus history
        requests.post(f"{API_URL}/api/sessions/{st.session_state.session_id}/clear-temp-docs")
        st.session_state.indexed_files.clear()
        st.rerun()
    except Exception as e:
        st.sidebar.error(f"Gagal membersihkan dokumen: {e}")
```

Tambahkan endpoint baru di backend:
```python
@router.post("/{session_id}/clear-temp-docs")
def clear_temp_docs(session_id: str):
    # Hanya reset pipeline cache sesi ini
    if session_id in _session_pipelines:
        del _session_pipelines[session_id]
    return {"status": "cleared"}
```
