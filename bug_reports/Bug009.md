## Title
Session Pipeline Cache (`_session_pipelines`) Tidak Memiliki Eviction — Potensi Memory Leak

## Severity
🟡 **Medium**

## Priority
🟡 **P3 — Fix Before Production**

## Description

Dictionary `_session_pipelines` di `chat_service.py` menyimpan instance `RAGPipeline` untuk setiap sesi yang pernah menggunakan fitur temporary document. Pipeline ini tetap di memori selamanya kecuali:
- Server di-restart
- `clear_session_pipelines()` dipanggil (hanya terjadi saat dokumen global berubah)
- Session di-delete manual

Dalam lingkungan production dengan banyak pengguna, ini menyebabkan akumulasi memori yang tidak terkontrol. Setiap pipeline dengan dokumen yang diunggah bisa memakan 50–200MB memori.

## Steps to Reproduce
1. 100 pengguna berbeda membuat sesi dan mengupload dokumen.
2. Tidak ada yang mendelete sesi mereka.
3. Server memori meningkat ke 5–20GB.
4. OOM / server crash.

## Expected Result
Pipeline yang tidak digunakan dalam X waktu dihapus otomatis dari cache.

## Actual Result
Tidak ada batas atau eviction. Cache tumbuh tanpa batas.

## Recommendation

```python
import time
from collections import OrderedDict

MAX_SESSIONS = 100
SESSION_TTL = 3600  # 1 jam

_session_pipelines: dict = {}
_session_last_access: dict = {}

def get_session_pipeline(session_id: str, base_pipeline: RAGPipeline) -> RAGPipeline:
    # Evict oldest jika melebihi limit
    if len(_session_pipelines) >= MAX_SESSIONS:
        oldest = min(_session_last_access, key=_session_last_access.get)
        del _session_pipelines[oldest]
        del _session_last_access[oldest]
    
    if session_id not in _session_pipelines:
        sess_pipeline = RAGPipeline()
        sess_pipeline.load_or_build()
        _session_pipelines[session_id] = sess_pipeline
    
    _session_last_access[session_id] = time.time()
    return _session_pipelines[session_id]
```
