## Description

Laporan QA sebelumnya (TestCases.md, APITesting.md, FunctionalTesting.md) menyebut threshold RAG sebagai `0.15`. Namun kode aktual `core/rag.py` menggunakan `0.2` sebagai default.

**Kode aktual `core/rag.py`:**
```python
def search(self, query: str, top_k: int = 3, threshold: float = 0.2):
    #                                                         ↑ 0.2, bukan 0.15
```

**Kode aktual `core/llm.py` (saat memanggil search):**
```python
results = self.rag_pipeline.search(message, top_k=3, threshold=0.15)
#                                                              ↑ 0.15 override
```

**Penjelasan:** `llm.py` memanggil `search()` dengan `threshold=0.15` secara eksplisit (override default). Jadi threshold efektif untuk chat adalah **0.15**, bukan 0.2.

Ini bukan bug di kode, tapi **dokumentasi QA yang tidak akurat** — nilai yang disebutkan kebetulan benar untuk chat flow, tapi penjelasannya misleading karena menyebut "default".

## Impact

Rendah — fungsionalitas tidak terpengaruh. Hanya dokumentasi QA yang perlu dikoreksi.

## Recommendation

Update semua dokumentasi QA untuk mencantumkan:
- **Default threshold `RAGPipeline.search()`:** `0.2`
- **Threshold efektif untuk chat (`llm.py`):** `0.15` (explicit override)
- **Threshold efektif untuk dev test scripts:** `0.1` (dari `scratch_rag_test.py`)
