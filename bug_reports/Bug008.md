## Title
Orphaned User Message di Database Jika Streaming Terputus di Tengah Jalan

## Severity
🟠 **High**

## Priority
🟠 **P2 — Fix in Next Sprint**

## Description

Dalam endpoint streaming `/api/chat/{session_id}/stream`, pesan user disimpan ke DB **sebelum** streaming dimulai. Jika koneksi terputus atau error terjadi saat streaming:
1. User message tersimpan di DB ✅
2. Assistant message **tidak** tersimpan (error menginterupsi `event_generator`) ❌
3. Hasil: DB memiliki user message tanpa assistant response yang sepadan

Pada pemanggilan berikutnya, history yang direkonstruksi dari DB akan mengandung urutan tidak valid: `user → user` (dua user message berturutan tanpa assistant di antaranya). Ini dapat menyebabkan perilaku LLM yang tidak terprediksi.

## Steps to Reproduce
1. Kirim pesan chat via streaming.
2. Putus koneksi jaringan saat streaming berlangsung.
3. `GET /api/chat/{session_id}/history` → lihat user message tanpa assistant response.
4. Kirim pesan baru → LLM menerima history yang malformed.

## Expected Result
Jika streaming gagal, user message juga tidak tersimpan (atomic operation).

## Actual Result
User message tersimpan, assistant message tidak. History DB menjadi malformed.

## Recommendation

**Option A — Simpan setelah streaming selesai:**
```python
# Jangan simpan user message sebelum stream
# Simpan keduanya setelah stream berhasil selesai
def event_generator():
    full_response = ""
    try:
        for chunk in chatbot.send_message_stream(chat_req.message):
            full_response += chunk
            yield chunk
        # Atomic save setelah berhasil
        with SessionLocal() as db_s:
            db_s.add(DBMessage(session_id=session_id, role="user", content=chat_req.message))
            db_s.add(DBMessage(session_id=session_id, role="assistant", content=full_response))
            db_s.commit()
    except Exception as e:
        yield f"\n[ERROR: Terjadi kesalahan, pesan tidak disimpan]"
```

**Option B — Cleanup on failure:**
Jika error, hapus user message yang sudah tersimpan dalam blok `except`.
