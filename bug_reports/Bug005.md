---

## Description

**v1.0:** History tumbuh di RAM tanpa batas.  
**v2.0:** History kini disimpan di SQLite (RAM tidak lagi jadi masalah), TAPI setiap request merekonstruksi seluruh history dari DB dan mengirimnya ke Groq API.

**Kode aktual v2.0:**
```python
# chat_service.py — send_chat_message()
for msg in db_messages:  # ← SEMUA pesan dari DB
    chatbot.history.append({"role": msg.role, "content": msg.content})

# Lalu di llm.py:
messages_to_send = [msg.copy() for msg in self.history]  # ← Semua dikirim ke Groq
```

Di sesi 100-turn: ~200 pesan (~20.000 token) dikirim ke API setiap request.

**Yang sudah Fix:** ✅ Memory RAM tidak lagi jadi masalah  
**Yang masih Open:** ❌ Token cost & latency meningkat di sesi panjang  

**Re-test Status:** ⚠️ **PARTIAL — Memory fixed, token growth masih open**

---

## Recommendation

Implementasi sliding window saat rekonstruksi history:
```python
# Di chat_service.py:
MAX_HISTORY_TURNS = 20
recent_messages = db_messages[-(MAX_HISTORY_TURNS * 2):]  # 20 pasang user+assistant
for msg in recent_messages:
    chatbot.history.append({"role": msg.role, "content": msg.content})
```
