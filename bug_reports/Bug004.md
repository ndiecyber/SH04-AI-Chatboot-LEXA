## Description

`RateLimitError` dari Groq API tidak ditangani secara spesifik. Generic `except Exception as e` menangkap semua error dan menghasilkan pesan teknis yang tidak ramah user.

**Kode aktual v2.0:**
```python
except Exception as e:
    self.history.pop()
    raise RuntimeError(f"Gagal memproses request ke Groq API: {e}")
    # ← 429 RateLimitError tertangkap di sini tanpa handling khusus
```

**Re-test Status:** ❌ **FAIL — Bug Still Open**

---

## Recommendation

```python
from groq import RateLimitError, AuthenticationError

try:
    ...
except RateLimitError:
    self.history.pop()
    raise RuntimeError("Sistem sedang sibuk. Silakan tunggu beberapa detik dan coba lagi.")
except AuthenticationError:
    self.history.pop()
    raise RuntimeError("API Key tidak valid. Periksa konfigurasi .env Anda.")
except Exception as e:
    self.history.pop()
    raise RuntimeError(f"Gagal memproses request: {e}")
```
