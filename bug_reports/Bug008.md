## Description

Di endpoint streaming, user message disimpan ke DB **sebelum** streaming dimulai. Jika koneksi putus atau error terjadi mid-stream, user message tersimpan tapi assistant message tidak — mengakibatkan history `user → user` yang malformed.

**Kode aktual `chat.py`:**
```python
# User message disimpan SEBELUM stream (baris ~65)
db.add(db_user_msg)
db.commit()   # ← sudah committed ke DB

def event_generator():
    try:
        for chunk in chatbot.send_message_stream(...):
            yield chunk
        # Assistant disimpan SETELAH stream selesai (baris ~85)
        with SessionLocal() as db_session:
            db_session.add(db_assistant_msg)
            db_session.commit()
    except Exception as e:
        yield f"\n[ERROR: {str(e)}]"
        # ← Tidak ada cleanup user message jika error di sini
```

**Efek samping tambahan:** `SessionLocal()` baru dibuat di dalam `event_generator()` — ini adalah **connection yang tidak di-manage oleh dependency injection FastAPI**, berpotensi connection leak jika generator tidak selesai dengan bersih.

**Re-test Status:** ❌ **FAIL — Bug Still Open** *(+ connection leak baru teridentifikasi)*

---

## Recommendation

Simpan kedua pesan secara atomik setelah stream selesai:
```python
def event_generator():
    full_response = ""
    try:
        for chunk in chatbot.send_message_stream(chat_req.message):
            full_response += chunk
            yield chunk
        # Atomic save setelah BERHASIL
        with SessionLocal() as db_s:
            db_s.add(DBMessage(session_id=session_id, role="user", content=chat_req.message))
            db_s.add(DBMessage(session_id=session_id, role="assistant", content=full_response))
            db_s.commit()
    except Exception as e:
        # Tidak ada yang disimpan jika gagal
        yield "\n[Maaf, terjadi kesalahan. Pesan tidak disimpan.]"
```
