## Description

Di `backend/routers/chat.py`, endpoint `/stream` membuat `SessionLocal()` baru secara manual di dalam `event_generator()`. Ini **tidak di-manage oleh dependency injection FastAPI** (`Depends(get_db)`).

**Kode aktual:**
```python
def event_generator():
    full_response = ""
    try:
        for chunk in chatbot.send_message_stream(chat_req.message):
            full_response += chunk
            yield chunk
            
        # ← SessionLocal() baru dibuat manual di sini
        from backend.database import SessionLocal
        with SessionLocal() as db_session:
            db_session.add(db_assistant_msg)
            db_session.commit()
            
    except Exception as e:
        yield f"\n[ERROR: {str(e)}]"
        # ← Jika exception terjadi, context manager 'with' mungkin tidak selesai
        #    tergantung di mana exception terjadi
```

**Risiko:**
1. Jika `event_generator()` di-cancel oleh client (disconnect), `with SessionLocal()` mungkin tidak pernah di-enter → connection tidak pernah dibuka (OK), TAPI jika exception terjadi setelah `with` di-enter dan di dalam exception handler tidak ada `yield` yang menyelesaikan generator, connection bisa hang.
2. `from backend.database import SessionLocal` di dalam fungsi adalah anti-pattern — seharusnya di-import di level modul.
3. Berbeda dengan endpoint sync yang menggunakan `db: Session = Depends(get_db)` dengan lifecycle yang dikelola FastAPI.

## Steps to Reproduce

1. Buka koneksi streaming di browser.
2. Tutup tab/browser sebelum streaming selesai (client disconnect).
3. Monitor jumlah koneksi SQLite aktif — `lexa.db` mungkin memiliki koneksi yang tidak tertutup.

## Expected Result

Koneksi database selalu ditutup dengan benar, bahkan saat client disconnect.

## Actual Result

Potensi koneksi SQLite tidak tertutup jika generator di-cancel mid-execution.

## Recommendation

**Option A — Import di level modul:**
```python
# Di bagian atas chat.py (bukan di dalam fungsi)
from backend.database import SessionLocal
```

**Option B — Gunakan try/finally untuk guaranteeing cleanup:**
```python
def event_generator():
    full_response = ""
    db_session = SessionLocal()
    try:
        for chunk in chatbot.send_message_stream(chat_req.message):
            full_response += chunk
            yield chunk
        db_session.add(DBMessage(role="assistant", content=full_response, ...))
        db_session.commit()
    except Exception as e:
        db_session.rollback()
        yield "\n[Maaf, terjadi kesalahan.]"
    finally:
        db_session.close()  # ← Selalu ditutup
```
